# 移动端适配优化简易指南 (Step-by-Step)

本指南旨在通过最简单、高效的操作，使 `frontend-vue` 项目具备良好的移动端访问体验。优化重点在于**响应式布局切换**和**基础交互调整**。

---

## 第一阶段：全局基础配置

### 1. 确认 Viewport 设置
确保项目根目录下的 [index.html](file:///home/devbox/project/frontend-vue/index.html) 包含以下代码：
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
```

### 2. 引入基础响应式工具类
在 [App.vue](file:///home/devbox/project/frontend-vue/src/app/App.vue) 的 `<style>` 中添加以下通用类：
```css
/* 移动端隐藏类 */
@media (max-width: 768px) {
  .hidden-mobile {
    display: none !important;
  }
}

/* 移动端间距缩减 */
@media (max-width: 480px) {
  .main-content {
    padding: 10px !important;
  }
}
```

---

## 第二阶段：核心框架适配

### 3. 侧边栏 (Sidebar) 响应式
修改 [App.vue](file:///home/devbox/project/frontend-vue/src/app/App.vue)：
- **操作**：在屏幕宽度小于 768px 时，通过 CSS 隐藏侧边栏。
```css
@media (max-width: 768px) {
  .aside-menu {
    position: absolute;
    z-index: 1000;
    left: -220px; /* 默认隐藏 */
    transition: left 0.3s;
  }
  .aside-menu.is-active {
    left: 0; /* 展开状态 */
  }
}
```

### 4. 头部导航 (Header) 简化
修改 [HeaderNav.vue](file:///home/devbox/project/frontend-vue/src/app/components/HeaderNav.vue)：
- **操作**：对不重要的按钮添加 `.hidden-mobile`。
- **目标**：全屏按钮、面包屑导航在手机端应隐藏。

---

## 第三阶段：页面内容适配

### 5. 栅格系统重构 (最重要)
在所有使用 `el-row` 和 `el-col` 的页面（如 [Dashboard.vue](file:///home/devbox/project/frontend-vue/src/app/views/Dashboard.vue)）中，将固定的 `span` 改为响应式属性。
- **修改前**：`<el-col :span="12">`
- **修改后**：`<el-col :xs="24" :sm="12">`
- **效果**：在手机（xs）上占满整行，在平板（sm）及以上并排。

### 6. 诊断页 (Diagnosis) 布局调整
修改 [Diagnosis.vue](file:///home/devbox/project/frontend-vue/src/app/views/Diagnosis.vue)：
- **操作**：将左右分栏的 `el-col` 调整为 `xs="24"`。
- **交互**：在移动端，建议将“日志预览”收纳进折叠面板或通过按钮切换显示，优先保证“诊断助手”的聊天体验。

### 7. 表格横向滚动
确保所有 [el-table](file:///home/devbox/project/frontend-vue/src/app/views/ClusterList.vue) 都包含在具有 `overflow-x: auto` 的容器中，或直接在 `el-table` 上设置 `min-width`。

---

## 第四阶段：交互细节微调

### 8. 弹窗宽度适配
搜索项目中所有的 `el-dialog`：
- **操作**：将 `width="50%"` 或固定像素宽度改为变量。
```html
<el-dialog :width="isMobile ? '95%' : '50%'">
```
- **脚本支持**：在组件中定义 `const isMobile = window.innerWidth < 768`。

### 9. 按钮与输入框
- **操作**：将移动端常用的操作按钮高度设为 `large` 或增加点击热区。
- **输入框**：确保 `type="textarea"` 的输入框在移动端不会因为软键盘弹起而遮挡。
