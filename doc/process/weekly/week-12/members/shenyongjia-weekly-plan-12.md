# 第十二周个人周计划（沈永佳）

## 目标概述
- 面向本周“α”版本交付窗口（至周三），完成“登录 + 权限管理”的最小可交付闭环
- 提供跨局域网可访问的后端与数据库部署方案，并完成最小验证与文档归档
- 准备统一格式的日志测试数据与访问路径，支撑后续日志查询与验证

## 本周职责
- 部署与远程访问方案验证（后端与数据库）
- 登录与权限最小闭环的后端接口联调与自检
- 统一配置与文档沉淀（`.env`、部署与排障说明）
- 与日志测试数据准备协同，提供后端读取验证示例

## 任务清单
- 部署与远程访问（跨局域网）
  - 设计访问拓扑与网络策略；确认 `listen_addresses`、`pg_hba.conf` 网段放行与 `5432` 端口开放
  - 编写最小验证步骤：本地/异地主机 `psql` 连接与后端 API 访问测试
- 后端认证与权限（最小闭环）
  - 统一 JWT 声明与角色校验；联调登录/注册/当前用户接口与受限路由访问
  - 错误码与提示口径对齐；完善异常路径与边界用例
- 配置与文档
  - 统一 `backend/.env` 与 `DATABASE_URL` 说明，补充 `backend/README.md` 的部署与排障步骤
  - 前端代理目标校验（`frontend-vue/vite.config.ts` 的 `VITE_API_TARGET`），联调指引补充
- 日志测试数据协同
  - 与数据侧确认 HDFS 样本路径与访问方式；提供后端读取示例与联调指引

## 时间安排
- 周一：部署与远程访问方案初稿与基础验证；整理网络与权限配置清单
- 周二：完成登录与权限最小闭环联调；补齐异常路径与边界用例
- 周三：交付“α”版本（登录 + 权限），完成跨局域网验证与文档归档
- 周四-周日：完善日志测试数据读取与指引；整理风险与后续优化项

## 验收标准
- 登录与权限：账号可登录、权限路由生效、错误提示清晰；联调用例通过
- 部署与访问：后端与数据库在跨局域网环境下可访问；提供部署与排障文档
- 日志数据：提供可读取的 HDFS 测试数据与路径说明，测试人员能完成验证
