# 第十四周小组周计划

## 核心目标
本周是项目的**冲刺周**，核心目标是实现系统功能的**全量闭环**与**深度联调**。重点解决集群连接不稳定性问题，打通 AI Agent 的工具调用链路，完成前端所有核心模块的接口对接，并开展系统性的功能测试与异常场景验证，确保系统具备交付状态。

## 详细计划

### 1. 后端与架构（沈永佳 & 李涛）
- **集群连接攻坚**：
  - 彻底解决 SSH 在跨网段环境下的超时与连接中断问题，优化连接池重试机制。
  - 引入连接健康度监控，确保指令下发的可靠性。
- **AI 工具链落地**：
  - 开发标准化的运维工具函数（如服务重启、日志检索），定义清晰的 JSON Schema。
  - 完善 `DiagnosisAgent`，实现 LLM 对工具的精准调用与结果解析，打通“诊断-修复”闭环。
- **接口收尾**：
  - 补全剩余的业务接口（高级监控、批量操作），统一全局异常处理与错误码规范。

### 2. 前端与交互（邢远鑫）
- **核心模块联调**：
  - **集群管理**：完成注册、启停、注销功能的接口对接，确保状态同步。
  - **故障诊断**：打通 AI 对话历史、故障摘要与日志预览功能。
  - **运维功能**：完成执行日志、告警配置、操作日志的 CRUD 联调。
- **体验优化**：
  - 统一全局加载态、禁用态与错误提示样式。
  - 完善账号管理与全局搜索功能。

### 3. 测试与质量（王祖旺）
- **测试体系构建**：
  - 梳理并执行全量功能测试用例，覆盖核心业务流程（日志->诊断->修复）。
  - 设计异常输入、网络抖动等边界场景测试用例。
- **质量验收**：
  - 引入测试覆盖率工具，量化代码质量。
  - 每日输出缺陷清单，督促开发人员及时修复，确保关键流程无阻断性 Bug。

## 风险与应对
- **网络环境限制**：若物理网络不可达，准备反向代理（FRP/Cloudflare Tunnel）作为备选方案。
- **AI 幻觉风险**：严格限制 Agent 工具执行权限，关键操作引入人工确认机制。
- **联调进度阻塞**：设立每日站会，快速拉齐前后端接口差异，优先解决阻塞性问题。
